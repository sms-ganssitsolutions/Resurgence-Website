VinAI Operating Principles (Strict)
When working on this project, always follow these rules:
1. Performance First
Public storefront must remain framework-less (vanilla HTML/CSS/JS).
Never suggest React, Next.js, Vue, or heavy libraries for the public shop or customizer.
Admin dashboard may use a framework, but aggressively optimize it (replace Framer Motion with CSS, use Server Components, etc.).
2. Mobile-Rigid UI
Every interactive element must use `touch-action: manipulation`.
Zero tap delay.
Test interactions on low-end Android devices mentally.
Kanban on mobile should become horizontal scroll + tap detail (no dragging).
3. Edge & Infrastructure
Prefer Cloudflare ecosystem (Pages, R2, Workers, KV, Cache Everything rules).
All logo uploads must go directly to R2 via presigned URLs.
Branding should be servable from edge (KV or lightweight API).
4. Branding & Aesthetic
Use the dynamic CSS variable system.
Company name: RESURGENCE
Tagline: POWERED BY DESIGNXPRESS
Energy: Competitive, high-performance, "crew that dominates".
Future marketing videos: 9:16 vertical, realistic cinematography, ring light, dynamic product flexing.
5. Currency
Supported: USD (base), HKD, PHP, SGD
All prices must update live when currency changes.
Quote payload must include `currency`, `currencySymbol`, and `totalInCurrency`.
6. Communication Style
Candid and direct. No fluff.
Use bold text for critical points.
Structure responses with clear headings and bullets.
When delivering code, provide complete, production-ready blocks.
Always point out performance implications.
